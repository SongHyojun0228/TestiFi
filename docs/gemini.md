# Gemini 수칙 (Gemini 3.1 Pro)

> Gemini가 TestiFi 프로젝트에서 따라야 하는 규칙.

## 프로젝트 컨텍스트

- **프로젝트명**: TestiFi (테스티피)
- **역할**: 콘텐츠 생성 + UI 다듬기 + QA
- **중요**: Claude Code가 아키텍처/핵심 로직 담당. Gemini는 보조. 아키텍처 임의 변경 금지.

## 핵심 역할 1: 심리테스트 콘텐츠 생성

Gemini의 가장 중요한 일은 **바이럴이 터지는 심리테스트 콘텐츠**를 만드는 것이다.

### 테스트 생성 절대 규칙

```
[필수] 질문 5~8개. 10개 넘기지 않는다.
[필수] 선택지 2~4개. 정답처럼 보이면 안 된다.
[필수] 질문은 "상황 제시 → 선택" 시나리오. 직접 성격 질문 금지.
[필수] 결과 유형 4~8개. 재미있고 공유하고 싶은 이름.
[필수] 결과 톤: 긍정 80% + 약간의 찔림 20%. 기분 나쁘면 공유 안 한다.
[필수] 궁합(best/worst) 반드시 포함.
[필수] 자연스러운 한국어. 번역투 절대 금지.
```

### 바이럴 테스트 황금 공식

```
1. 제목 패턴: "나의 _____ 유형은?"
   ✅ "2026 나의 연봉 유형 테스트"
   ✅ "나의 카페 주문 성격 테스트"
   ❌ "당신의 심층 심리 분석" (무겁다)

2. 질문 패턴: 일상 상황 + 의외의 선택지
   ✅ "금요일 퇴근 후 갑자기 3시간이 생겼다. 뭐 할래?"
   ❌ "당신은 외향적인 편인가요?" (직접 성격 질문 = 최악)

3. 유형명 패턴: 유머 + 공감 + 공유욕구
   ✅ "월급날만 기다리는 다람쥐형 🐿️"
   ✅ "조용한 플렉스 고수형 💎"
   ❌ "유형 A: 분석적 성격" (재미 0)

4. 결과 설명: "맞아 나 이런데!" 반응 유도
   ✅ "당신은 친구들 사이에서 갑자기 현실적인 말을 던져서 
       분위기를 싸하게 만드는 타입... 
       하지만 모두가 속으로는 인정하고 있어요. 🤭"
   ❌ "당신은 논리적이고 분석적인 성격입니다."

5. 궁합 표현: 구체적이고 재미있게
   ✅ "💚 찰떡궁합: 본능적 플렉서형 — 당신이 고민할 때 '그냥 사!' 해주는 존재"
   ✅ "💔 상극: 영수증 수집가형 — 커피값까지 나누자는 그 사람..."
```

### 시즌/트렌드 키워드

```
[봄 3-5월] 벚꽃, 새학기, 입사, 소개팅, 다이어트
[여름 6-8월] 여행, 바캉스, 더위, 축제, 휴가
[가을 9-11월] 독서, 감성, 추석, 연말 준비, 이직
[겨울 12-2월] 크리스마스, 연말결산, 새해계획, 겨울감성
[상시] 돈/연봉, 연애/궁합, MBTI 변형, 직장생활, 음식, 카페
```

## 핵심 역할 2: UI 다듬기

```
[필수] Claude가 만든 컴포넌트 구조 유지. 구조 변경 금지.
[필수] shadcn 컴포넌트 기반. 소스 직접 수정 금지 → className으로 오버라이드.
[필수] 모바일 퍼스트. 데스크탑은 보조.
[필수] 질문 페이지: 한 화면에 질문 1개. 스크롤 없이.
[필수] 결과 페이지: 공유 버튼이 스크롤 없이 보여야.
```

### Earthy Tones 디자인 규칙

```
배경:        #F5F0E8 (따뜻한 크림)
카드:        #FAF7F2 (밝은 크림) + subtle shadow
메인 강조:   #9B9B5A (올리브) — 프로그레스바, 활성 상태
보조 강조:   #C4967A (살몬) — 호버, 세컨더리 버튼
CTA/액센트:  #B57B86 (로즈) — 공유 버튼, 중요 CTA
텍스트:      #3D3632 (다크 브라운) — 가독성 확보
서브텍스트:  #8A7E76 (웜 그레이)
보더:        #DDD5C5 (라이트 크림)
```

### 카카오톡 공유 버튼은 예외

```
카카오 버튼 색상은 브랜드 팔레트 대신 카카오 공식 색상 사용:
  배경: #FEE500
  텍스트/아이콘: #191919
이유: 카카오 브랜드 가이드라인 준수 + 사용자 인식률
```

### 애니메이션

```
✅ 질문 전환: fade + translateY(10px→0) (300ms ease-out)
✅ 결과 공개: scale(0.95→1) + fade (200ms)
✅ 프로그레스바: width transition (400ms ease)
✅ 버튼 호버: 살짝 lift (translateY -2px) + shadow
❌ 무거운 라이브러리 (framer-motion, lottie 등)
❌ 과도한 바운스/스프링 (CLS 악화)
```

## 핵심 역할 3: QA/테스트

```
[필수] 카카오톡 인앱브라우저에서 정상 작동 확인
[필수] 삼성 인터넷 브라우저 호환성 확인
[필수] OG 이미지가 카카오 미리보기에서 제대로 뜨는지 확인
[필수] 100vh 이슈 대응 (dvh 또는 JS 실제 뷰포트 계산)
[필수] safe-area-inset-bottom 적용 (아이폰 하단바 대응)
```

## Claude와 협업 프로토콜

```
1. Claude가 컴포넌트 뼈대 → Gemini가 디자인/인터랙션 추가
2. Claude가 API 작성 → Gemini가 에러 핸들링/엣지 케이스 보강
3. Gemini가 콘텐츠 생성 → Claude가 스키마에 맞게 구조화
4. 충돌 시 → Claude의 아키텍처 결정이 우선
```
